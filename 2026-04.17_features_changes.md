# Session Summary & Feature Changes (feature/fix-views)
**Date**: 2026-04-18
**Branch**: `feature/fix-views`

## ELI5: What happened this session?
We had a "perfect storm" of small technical glitches that made the application look dead.
1. **The Language Barrier**: The AI service and the Web service were speaking slightly different versions of the same language (gRPC/Protobuf), causing them to ignore each other.
2. **Missing Instruction Manuals**: The system was looking for a "manifest" (a list of all visual files) that didn't exist because a specific helper (Laravel Vite Plugin) wasn't installed.
3. **The ID Card Mismatch**: The database was expecting "Number" IDs, but the code was handing out "Random Text" IDs (ULIDs). The database simply refused to save them.
4. **The Security Guard**: The login system was too strict. It didn't recognize `localhost` as a safe place, so it kept kicking users out (401 error) even after they logged in correctly.

## Feature Changes & Improvements
- **Stable Builds**: Switched from Alpine to Debian for PHP to fix C++ compilation bugs in gRPC.
- **Frontend Sync**: Installed and configured `laravel-vite-plugin` to correctly link CSS/JS assets.
- **Database Alignment**: Fixed primary key types in `Diagram`, `DiagramSnapshot`, and `AuditLog` models to match the database schema.
- **Test Suite**: Created new feature tests for Login and Diagram management to catch these issues earlier next time.
- **Visuals**: Applied a "Futuristic Dark" theme to the Login and Dashboard views.

## Step-by-Step Logic to Solve Issues

### 1. Fix Build Errors (Exit 255)
**Logic**: Alpine Linux lacked specific C++ headers for gRPC. 
**Command**: 
```bash
# Change FROM php:8.3-fpm-alpine to php:8.3-fpm in Dockerfile
# Add zlib1g-dev for compilation
```

### 2. Fix "Exec Format Error"
**Logic**: A blank line before `#!/bin/sh` in the entrypoint script confused the Linux kernel.
**Action**: Removed the leading blank line.

### 3. Fix 500 Database Errors
**Logic**: The code used `HasUlids` but the database used `bigint`.
**Fix**: Remove `HasUlids` trait and use a manual `boot` hook to generate ULIDs for the secondary column.
```php
static::creating(fn ($m) => $m->ulid = (string) Str::ulid());
```

### 4. Fix 401 Unauthorized
**Logic**: Sanctum needs to know which domains are "first-party" to allow cookie sharing.
**Commands**:
```bash
# Update .env.local
SANCTUM_STATEFUL_DOMAINS=localhost:8000,127.0.0.1:8000
# Clear cache inside container
php artisan optimize:clear
```

### 5. Fix 422 Validation Error
**Logic**: The HTML form sent `er` but the server only accepted `erd`.
**Action**: Updated `create.blade.php` select options to match the controller validation.
